## Overview

**AbDo-CTRG** introduces two contributions to address the gap in abdominal CT report generation:

- **AbDo-CT** — A curated abdominal CT dataset of 32,060 slices from 88 patients, each paired with expert-verified slice-level and patient-level radiology reports, plus quantitative organ volume measurements across 7 abdominal organs.
- **AbDo-RG** — A three-stage vision–language baseline that integrates a Vision Mamba encoder, CT–text contrastive alignment, and LLM-based diagnostic report generation.

| Avg. patient-wise report length | 166.5 words |
| Train / Val / Test split | 22,254 / 3,010 / 6,891 slices |

### Annotation Coverage

AbDo-CT provides annotations at two levels:

- **Patient-wise reports** — manually authored and verified by senior clinical specialists (gastroenterologist, hepatologist, urologist)
- **Slice-wise reports** — a gold standard subset (11,702 slices, 35 patients) with full manual annotation, and a verified silver standard (20,358 slices) generated by MedGemma and reviewed via human-in-the-loop verification

## Method: AbDo-RG

AbDo-RG is a progressive three-stage vision–language framework for CT report generation.

```
Stage 1: Autoregressive Pre-training (Vision Mamba)
           ↓
Stage 2: CT–Text Contrastive Alignment (RadBERT)
           ↓
Stage 3: Diagnostic-Guided Supervised Fine-Tuning (LLM + classification branch)
```

### Stage 1 — Autoregressive Visual Pre-training
- Backbone: **VMamba** (linear O(N) complexity vs. O(N²) for Transformers)
- CT slices are partitioned into 32×32 non-overlapping patches → projected to 1024-dim visual tokens
- Trained with an autoregressive reconstruction loss on unannotated CT volumes

### Stage 2 — CT–Text Contrastive Alignment
- Follows a CLIP-style contrastive objective
- Visual encoder: pre-trained VMamba from Stage 1
- Text encoder: **RadBERT** (domain-specific medical language model)
- Maps CT images and radiology reports into a shared semantic space

### Stage 3 — Diagnostic-Guided Fine-Tuning
- A cross-attention module bridges visual features into the LLM
- A parallel **pathology classification branch** (binary cross-entropy) adds diagnostic supervision
- Joint loss: `L_overall = L_gen + α · L_cls` (α = 5)
- LLM decoder: Qwen1.5-1.8B (for AbdomenAtlas-8K) or Llama-3.1-8B (for AbdomenAtlas 3.0)

## Getting Started

### Requirements

```bash
git clone https://github.com/floz-anonymous/AbDo-CTRG.git
cd AbDo-CTRG
pip install -r requirements.txt
```

### Data Preprocessing

```bash
python data/preprocessing/preprocess.py \
  --input_dir /path/to/raw_ct \
  --output_dir /path/to/processed \
  --spacing 1.0 1.0 4.0 \
  --hu_min -1000 --hu_max 1000
```

### Training

**Stage 1 — Autoregressive Pre-training:**
```bash
python train_stage1.py --config configs/stage1.yaml
```

**Stage 2 — Contrastive Alignment:**
```bash
python train_stage2.py --config configs/stage2.yaml \
  --pretrained_encoder checkpoints/stage1.pth
```

**Stage 3 — Supervised Fine-Tuning:**
```bash
python train_stage3.py --config configs/stage3.yaml \
  --pretrained_encoder checkpoints/stage2.pth \
  --llm qwen1.5-1.8b   # or llama-3.1-8b
```

### Evaluation

```bash
python evaluate.py \
  --checkpoint checkpoints/abdorg_best.pth \
  --dataset abdo_ct \
  --metrics bleu rouge meteor cider f1
```

## Annotation Protocol

AbDo-CT uses a dual-track annotation pipeline:

- **Track A (Slice-wise):** MedGemma-assisted generation with three-radiologist scoring and specialist escalation for conflicts. Inter-annotator agreement (Fleiss' κ = 0.751).
- **Track B (Patient-wise):** Organ-specialist authorship reviewed by a senior radiologist. Pairwise agreement κ = 0.815–0.842.

Full annotation was completed in ~84 days (vs. an estimated 200 days for fully manual annotation), a **6× speedup** for the silver standard phase.
